Strategic Approach

Primary Goal

Transform Alliance AMS from "zero online orders" to self-service catalog through limited product demonstration

Core Problem Solving

Alliance AMS customers currently face:

  • Cryptic part codes - "DBA4784A" means nothing to customers
  • No vehicle fitment info - "Will this fit my Hilux?" requires phone calls
  • Missing specifications - Technical details hidden behind codes
  • Zero online orders - Website doesn't enable purchasing decisions

MVP Components

CSV-to-iTabs Pipeline

Transform supplier data into customer-friendly information with AI enhancement

Product Catalog Enhancement

Add iTabs to existing website with vehicle fitment and specifications

Smartlink Integration

Simple "i" indicators linking to organized product information

MVP Limitations

Focus on 8-12 DBA brake products only. No images in V1. No WordPress plugin development.

Technical Implementation

Technology Stack

HTML5 CSS3 JavaScript JSON Data AI Enhancement Mobile-First

Architecture Overview

``` // Basic iTabs Structure i // JavaScript handles popup function showProductInfo(productCode) { // Fetch product data from JSON // Display in organized tabs // Vehicle fitment, specs, pricing }
```

Data Structure

Each product will have structured JSON data containing:

  • Vehicle Fitment - "Fits 2018+ Toyota Hilux 2.8L Diesel"
  • Specifications - Rotor diameter, thickness, material
  • Features - DBA technologies (XG-150, Kangaroo Paw, etc.)
  • Pricing - Retail/trade prices with stock status
  • Applications - Street, performance, racing usage

Integration Method

Minimal invasive approach:

  1. Add single JavaScript file to Alliance AMS website
  2. Add "i" indicators to selected product codes
  3. No database changes required
  4. Works with existing WordPress structure

Technical Benefits

Lightweight implementation that enhances rather than replaces existing website
